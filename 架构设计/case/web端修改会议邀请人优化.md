# web端修改会议邀请人优化

- **来源日期：** 20260324
- **类型：** 架构设计 / 案例

---

## 背景

会议预订、修改时，大会场景下邀请人可能超过千人。

## 问题现象

CGI → Center 的 RPC 协议基于 **UDP**，邀请人列表数据量过大时，导致 **UDP 包体超限**。

## 根因

UDP 协议对单个数据包大小有限制（通常 64KB），千人邀请列表序列化后的体积轻易超出限制，导致数据传输失败。

## 优化方案：引用分离

![修改会议邀请人优化](../../asset/修改会议邀请人优化.png)

新增一个**邀请人服务**：

```
原来：
CGI --[UDP, 携带完整邀请人列表]--> Center（包体超限 ❌）

优化后：
CGI --[HTTP]--> 邀请人服务（存储完整邀请人列表，返回临时 ID）
CGI --[UDP, 携带临时 ID]--> Center（包体小，不超限 ✅）
Center --[需要时]--> 邀请人服务（通过 ID 取回邀请人列表）
```

## 设计模式

此方案使用了「**引用分离**」设计模式，详见：[引用分离知识点](../knowledge/引用分离.md)

## 结论

通过将大体积的邀请人列表从主链路中剥离，用轻量的临时 ID 替代，解决了 UDP 包体超限问题，同时不影响业务语义。
