# 战术：冗余进攻

> 本页是 [[concepts/战术_联攻]]（联攻总论）的**子分类——冗余战术**。

## 核心前提

> **进攻冗余** = 我方存在**大 C**（能推队或至少 1 换 2），但被敌方某只宠物 counter / check 导致无法启动。需要创造额外进攻手段来突破这个关键拦截位。
>
> **冗余的前提条件**：存在 **1看2** 或 **2看3** 的重复对位局面。如果是 1看1、2看2，无法实施冗余战术 → 应转向 [[concepts/战术_非冗余]]。

## 一、定向施压战术

### 前提条件

我方有**复数个大 C**，且被敌方的**同一只（或少数几只）宠物** counter / check。

### 机制：1 看 2

```
我方：大C-A + 大C-B    vs    敌方：唯一 counter 位 X

X 能 counter A，也能 counter B → 但 X 只有一只，只能同时防一个

战术：
  上 A → 敌方只能上 X → A 硬打，虽然 A 最终会死，但 X 大残
  再上 B → X 已残，无法阻挡 B → B 推队 ✓
```

### 实战案例：白金 + 蹦床 vs 雪影娃娃

| 步骤 | 操作 | 结果 |
|------|------|------|
| 1 | 白金独角兽上场 | 敌方换雪影娃娃（唯一能 counter 的） |
| 2 | 白金硬打 | 白金死，雪影大残 |
| 3 | 蹦床松鼠上场 | 雪影已残，无法阻挡 → **蹦床推队** |

**角色定义**：
- **进攻冗余目标**：雪影娃娃（需要被突破的 counter 位）
- **起点**：白金独角兽（牺牲位，消耗目标状态）
- **终点**：蹦床松鼠（获益者，突破后轻松推队）

⚠️ 起点和终点**可以互换**——先上蹦床消耗、后上白金推队也成立。

### 机制扩展：2 看 3

```
我方大C ×3：黑猫巫师 + 白金独角兽 + 蹦床松鼠
敌方拦截位 ×2：
  - 圣剑-X → counter 黑猫 + 白金
  - 雪影娃娃 → counter 白金 + 蹦床

战术：上白金（被任一拦截位 counter），硬打消耗掉圣剑或雪影
     → 场下黑猫/蹦床无人可挡
```

### 定向施压的优劣势

| 维度 | 评估 |
|------|------|
| ✅ 灵活性 | 起点/终点可互换，对手难以预判先上谁 |
| ✅ 威慑力 | 多 C 阵容本身具有强大场上压力 |
| ❌ 阵容代价 | 复数大 C 往往意味着放弃防守/辅助能力，给敌方可乘之机 |

## 二、控制战术

### 与定向施压的核心区别

| 对比项 | 定向施压 | 控制战术 |
|--------|----------|----------|
| **阵容要求** | 复数个大 C | 1 个大 C + 控制手 |
| **起止点关系** | 起点/终点**可互换** | 起点/终点**固定不可换** |
| **灵活性** | 高 | 较低 |
| **适用场景** | 进攻型阵容 | 相对平衡型阵容 |

### 机制

```
我方：大C（蹦床松鼠）+ 控制手（雪影娃娃）
敌方：counter 位（燃薪虫）— 同时 counter 大C 和控制手

战术：
  反复切换雪影娃娃上下场 → 给燃薪虫叠加冻结状态
  → 逐渐处理掉燃薪虫 → 大C 蹬床松鼠有机会推队
```

**固定角色**：
- **起点（必须）**：控制手（雪影娃娃）
- **终点（必须）**：大 C（蹦床松鼠）
- 不能互换——控制手无法推队，大 C 无法独立处理 counter

## 与 Check/Counter 的关系

[[concepts/战术_对位关系]] 定义了 check/counter 的基本形式，而**冗余战术是当 counter 关系形成瓶颈且存在重复对位时的系统化突破方案**。

## 阵容构建启示

1. **如果主 C 有明确 counter 位**，考虑是否配备"冗余起点"（另一个大C 或强力控制手）
2. **定向施压路线**：双 C 或三 C 阵容，牺牲防守换突破力
3. **控制路线**：单 C + 辅助/控制，更平衡但依赖控制手的生存能力
4. **识别敌方拦截结构**：对方用几只 pet 来拦你的几个 C → 判断属于 1看N 还是 M看N 局面
